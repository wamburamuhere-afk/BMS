Headline change

The "System requires your attention" panel, rebuilt to be trustworthy

The dashboard's yellow alert panel — "You have N pending notifications" — was showing everything at once, some of the numbers were not real, and clicking an item often dropped you on a whole list instead of the thing that needed attention. It has been rebuilt from the ground up so that every number is true, and every click takes you straight to the work.

1. Tidy, grouped view — open only what you need

Clicking View Details now opens a neat, collapsible list of categories (Inventory & Products, Invoices, Goods Receipt Pending, and so on), each showing a count of items that need attention — nothing is dumped on screen until you ask for it.

2. Two clear actions on every category

  • View here — expands the list of items right on the dashboard, exactly as before.
  • Go to source — jumps to that module's real page, filtered to show only the items that need attention — the same number you saw on the badge.

3. Every item opens the exact record

Previously, clicking an alert could send you to a general list. Now each item goes straight to the right place:

  • Goods Receipt Pending → opens the Receive-Goods screen for that exact purchase order.
  • Pending leave → opens that specific leave; quotations and tenders → their own record.
  • Over-credit customer → that customer; negative stock → that product; overdue invoice / low stock → their detail view.
Before
A wall of mixed alerts, some numbers inflated or stale, and links that landed you on a full list to hunt through.
Now
Grouped counts you can trust, a filtered "Go to source" for each, and per-item links that open the exact record.
Self-correcting by design. Because the counts are now live readings of the actual data (not saved messages), the moment you fix the underlying issue — restock an item, receive goods, renew a document — it disappears from the count on its own. No manual clearing needed.

Accuracy & reliability

Three hidden miscounts, found and corrected

Building the filtered views exposed three genuine counting problems that had been quietly under- or over-stating the figures. All three are now fixed and consistent.

  • Overdue invoices were undercounted. The system automatically re-labels past-due invoices as "overdue", but the dashboard was only counting a narrow set of labels — so re-labelled and "reviewed" invoices silently vanished from the number. Overdue is now defined one consistent way everywhere: not paid, cancelled or draft, past the due date, with a balance. The real figure rose from 1 to 6.
  • Customer credit exposure used the wrong basis. A customer's "amount owed" now uses the true outstanding balance (invoice total minus what's been paid) across all unpaid invoices — so partial payments count correctly and no over-limit customer is missed.
  • "Document expiry" was counting stale reminders. The badge showed 13 based on old, unread reminder messages — but only 1 document is actually expiring. It now reads the document library directly, so it shows the real number and updates itself when a document is renewed.
What this means for you: the number on the alert panel is now a number you can act on. If it says 6, there are 6 — and clicking through shows you those exact 6.
Inside a project

Sales, Finance and Notes screens grew up

Several sections inside a project's screen were upgraded from basic lists into full, searchable, sortable tables — with a card view that automatically takes over on phones.

  • Sales section — Sales Orders, Interim Payment Certificates (IPC) and Invoices became proper tables with search, sorting and paging, plus a tidy mobile card view.
  • Finance section — Expenses, Payment Vouchers and Budget items received the same treatment: searchable tables on desktop, clean cards on mobile.
  • NIP list — an Item Code column was added so the in-project list matches the main page.

Real Project Notes (replacing a mock-up)

The project Notes tab used to be a dummy — "Add Note" only showed a success message and saved nothing. It is now a real, working notes feature:

  • Notes are genuinely saved, listed, and can be managed.
  • Professional filtering, pagination and appearance, in line with the rest of the system.
  • A proper print header (company name, project, contract number) so a printed notes sheet looks official.
Printing

Everything now prints the way it looks on screen

Several project screens printed badly — the data came out as a narrow strip of phone-style cards squeezed to the left with an empty page beside it, and some sections had no header at all. The printing was corrected across the board.

  • Sales Orders, Invoices, IPC and the Finance tables (Budget, Vouchers, Expenses) now print the full wide table — not the phone cards — with the on-screen "Actions" buttons hidden and no more blank extra page.
  • Bills and Inventory tabs gained the professional print header (logo, company name, project, contract number) the other tabs already had.
  • The Budget print no longer shows the little empty expand column.
  • Warehouse "View Details / Stock History" — the header used to repeat on every page and push the content onto page 2. It now appears once at the top of page one, with the content right beneath it.
Smoother, safer data entry

Fewer dead-ends when adding records

  • Project expenses — the Expense Type list now respects which types belong to projects (Admin, Fixed, Operating are available and the field is optional), instead of a hard-coded list; and saving an expense now always tells you if something fails, rather than failing silently.
  • Products — a zero price no longer blocks saving, the form always reports the outcome, and duplicate SKU / product codes are caught up front.
  • Sales orders — the salesperson is now set automatically to the logged-in user on create and edit.
  • Navigation — a clean "Back to Project" path was added for Supplier, Sub-contractor and Staff screens, and a dead, unused staff wizard was removed.
