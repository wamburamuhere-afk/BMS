Suppress Duplicate Company Header When Printing Project Reports
Bug Fix
app/bms/operations/project_view.php header.php
Problem: When printing from Project Details tabs (Milestones, Reporting, Reports, Docs), the company logo and name appeared twice — once from the global header and once from the tab's own print header.
Fix: Added PHP constant define('BMS_SUPPRESS_PRINT_HEADER', true) at the top of project_view.php. Modified header.php to check for this constant before rendering the global print header. Constants work across PHP function scope boundaries unlike variables — this was the key to making it work.

Project Details — Mobile Card View for Milestones, Reporting & Reports Tabs
Bug Fix
app/bms/operations/project_view.php
Problem: The Milestones, Reporting, and Reports tabs inside Project Details were not switching to card view on mobile. These tables use custom AJAX rendering (not DataTables), so the global bms-mobile-cards.js init.dt hook never fired for them.
Fix: Added renderForTable() calls at the end of each table's render function — loadMilestones(), renderReportingTable(), and loadPerformanceData(). Also hid hardcoded toggle buttons on mobile using a CSS media query with attribute selector.

Mobile Card Left/Right Spacing Reduction
Fix
assets/css/responsive.css
Problem: On mobile, card view had excessive left and right margins (~0.64cm), leaving cards too narrow and not using screen space effectively.
Fix: Added @media (max-width: 767px) override to set .bms-auto-cards padding to 4px and card item left/right padding to 4px. Effective margin reduced from ~0.64cm to ~0.32cm. Applies globally to all card views.

Milestone Card Auto-Refresh After Add / Delete
Fix
app/bms/operations/project_view.php
  • After adding a new milestone row, the card view was not updating to show the new entry.
  • After deleting a milestone row, the card view still showed the deleted entry.
  • Added renderForTable('milestonesTable') at the end of both addNewMilestoneRow() and removeMilestoneRow() functions.

Critical Bug — Desktop Defaulting to Card View (noStore Fix)
Critical Bug Fix
assets/js/bms-mobile-cards.js
Problem: Visiting any page on mobile caused the system to auto-switch to card view and save 'card' to localStorage. The next time the same page was opened on desktop, it read 'card' from localStorage and defaulted to card view — with no way to know why or reset it.
Fix: Added a third noStore boolean parameter to the toggleAutoView() function. All mobile auto-switches now pass true to skip saving. Only explicit user clicks on desktop toggle buttons save the preference to localStorage.

Milestone Edit Modal for Mobile Card View
New Feature
app/bms/operations/project_view.php
  • On mobile, milestone editing uses hidden inline table rows — impossible to access when in card view since the table is hidden.
  • Added #milestoneEditModal — a full-screen modal (on small phones) with fields for Description, Unit, Scope (%), and Weight (%).
  • Weight field is automatically hidden for parent milestone rows (their weight is auto-calculated from children).
  • Modified editMilestoneRow() to detect mobile: on mobile it pre-fills and opens the modal; on desktop it keeps existing inline edit behaviour.
  • New saveMilestoneEditModal() function writes modal values back to the hidden table row, triggers recalculations, refreshes card view, and closes modal.

Fix

Feature + Fix
app/bms/operations/project_view.php

Performance Reports — Weekly/Monthly/Quarterly/Yearly Data Hidden on Right
Bug Fix
app/bms/operations/project_view.php
Problem: When switching from Daily to Weekly, Monthly, Quarterly, or Yearly report view, columns on the right side of the table were cut off and not visible. The table data appeared hidden on the right-hand side.

11
Performance Report Card View — Broken for Non-Daily Periods
Bug Fix
assets/js/bms-mobile-cards.js
Problem: On mobile, switching to Weekly/Monthly/Quarterly/Yearly report view showed completely wrong content in card view — S/NO numbers appearing as card titles, wrong field labels, and no action buttons. Daily view worked correctly.
Root Cause: The non-daily view generates a 2-row table header. enhancePlainTable() was using thead tr:last-child th which picked up only the 3 print-only sub-headers in the second row instead of the correct 7 column headers in row 1. Also, non-daily table rows include 3 hidden print-only cells (d-none) that misaligned column indices.

Fix: Changed header extraction to thead tr:first-child th with a filter to skip d-none elements. Changed buildCard() to filter out d-none table cells before building cards.
12
Print Output Showing Both Table and Card View Together
Bug Fix
assets/js/bms-mobile-cards.js
Problem: When printing while in mobile card view, the print output showed both the data table AND the card view together — making reports look mixed and unprofessional.
Fix: Added d-print-none class to card container divs at the point of creation in both enhanceTable() and enhancePlainTable(). Card containers are now always hidden during print regardless of current view mode or browser CSS rule ordering.
13
Toggle Buttons Appearing in Tab Navigation Row
Bug Fix
assets/js/bms-mobile-cards.js
Problem: Table/card view toggle buttons were incorrectly appearing inside the "Overview · Scope · Planning · Sales · Procurements · Finance · Notes · Staff · Docs · Reports" tab navigation bar in Project Details — ruining the visual appearance of the tab row.
Root Cause: injectToggleButtons() calls tableEl.closest('.card') which climbed up to the outer workspace card whose card-header holds the tab navigation (.nav-tabs).

Fix: Added a two-line guard — before injecting, check if the found card's header contains .nav-tabs or ul.nav. If yes, skip injection. Toggle buttons still correctly appear in inner content cards.

Purchase Orders, Products & Tenders — Mobile Card View Fixes
Bug Fix
app/bms/purchase/purchase_orders.php app/bms/product/products.php app/bms/tenders/tenders.php
  • Purchase Orders: Page was opening in card view on desktop. Cause — mobile auto-switch always saved 'card' to localStorage, desktop then read it. Fix: Wrapped localStorage.setItem() in a desktop-only guard (window.innerWidth >= 768).
  • Products: Mobile was not auto-switching to card view — page loaded in table view on all devices. Fix: Added mobile detection on page load and forced card view on mobile without saving to localStorage.
  • Tenders: Tenders table uses custom AJAX loading (not DataTables) so bms-mobile-cards.js never detected it. Fix: Added renderForTable('tenderTable') inside the loadTenders() success callback after each data render.
