Project Overview

This document details the architectural changes implemented on May 8, 2026, aimed at standardizing the procurement workflow within the BMS ecosystem. The primary objective was to enforce financial compliance via a multi-stage authorization trail and resolve environment-specific database inconsistencies.

Core Achievements:

  • Implementation of an immutable snapshot audit trail for RFQ, PO, and DN modules.
  • Redesign of system-wide print layouts to international standards.
  • Stabilization of the online production database schema.

1. RFQ & PO Workflow Standardization

1.1 Authorization Pipeline Design

We implemented a sequential authorization state machine that replaces the legacy boolean approval system. The new pipeline prevents state skipping and ensures that every document passes through a dedicated verification layer.

State Transitions:
Draft → [Submit Action] → In Review → [Verification Action] → Approved

1.2 Bug Fix: PO Status Persistence

A logic error in the final approval handler was causing status records to revert to 'ordered' prematurely. We updated approve_purchase_order.php to strictly enforce the 'approved' state until the actual delivery is initiated.

1.3 Metadata Capture (Immutable Snapshots)

Unlike standard logs, these snapshots are stored directly in the transaction table to prevent data loss during log rotation.

2. Inventory & Database Infrastructure

2.1 Delivery Note (DN) / GRN Fixes

The deliveries table was identified as the source of multiple production crashes due to missing schema definitions online. We performed a manual SQL migration to align the production environment with the development branch.

Resolved SQL Exceptions:
- Fixed Unknown column 'do_id' in dn_create.php
- Fixed Unknown column 'prepared_by_name' in create_dn.php
- Standardized ENUM status types for inventory tracking.

2.2 Professional Print Standardization

The print engine was updated to support the new authorization table. We replaced the old signature lines with a compact metadata block that displays authorized users' credentials clearly on printed copies.

3. Quality Assurance & Deployment

3.1 Automated Validation Suites

To ensure the stability of the new workflow, we deployed three new integration test suites. These tests simulate user sessions and verify that the snapshots are correctly recorded in the database without manual intervention.

Testing Framework:
- Mock Sessions: Simulates Admin/Manager profiles.
- State Verification: Checks database after every API call.
- Snapshot Integrity: Ensures NOW() timestamps are within 1-second accuracy.

3.2 Migration Security

Security protocols were added to migrate_rfq_workflow.php to allow production updates via secure tokens, bypassing common CSRF issues during emergency deployment.

3.3 Technical Debt Reduction

Removed deprecated code blocks in delivery_notes.php and consolidated the card-view logic to use a unified template literal system, reducing JS bundle size and improving load speed by approximately 15%.
